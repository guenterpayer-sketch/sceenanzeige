REST-API
Die Nimbuscloud-API.
Organisatorische Informationen
Schnittstellen — Die Nimbuscloud ist über die JSON-Schnittstellen (Current und Legacy) erreichbar. Beide werden laufend weiterentwickelt und gepflegt, neue Softwaremodule erhalten jedoch nur Endpunkte in der aktuellen API.
Legacy-Dokumentation — Die Dokumentation der Legacy-API findest du hier.
API-Keys — Sofern der Zugriff auf die API im gewählten Paket enthalten ist, können API-Keys in der Nimbuscloud unter System > Einstellungen > API-Keys erstellt werden.
IP-Beschränkung — API-Keys können aus Sicherheitsgründen auf bestimmte IP-Adressen eingeschränkt werden. Ist keine Einschränkung gewünscht, kann der Zugriff mit einem Stern (*) von jeder beliebigen IP-Adresse erfolgen.
Undokumentierte Felder — Felder, die über diese Dokumentation hinaus übertragen werden, sind nicht offiziell verfügbar und können ohne Vorankündigung umbenannt oder entfernt werden.
Deprecation — Aufrufe an einen veralteten Endpunkt werden mit einer Warnung im Log markiert. Der Zeitpunkt der Abkündigung ist in der Dokumentation des jeweiligen Endpunkts angegeben.
Support — Technischer Support ist ausschließlich über Support-Ticket oder per E-Mail an support@nimbuscloud.at verfügbar und wird individuell abgerechnet.
Hinweis. Für die meisten Anwendungsfälle von Webseiteneinbindungen ist die Verwendung der iFrame-Schnittstelle zu empfehlen. Alle Infos zu den verfügbaren Widgets findest du in deiner Nimbuscloud unter System > Einstellungen > Online-Anmeldung.
Technische Grundlagen
API-Key-Header — Der API-Key muss im Header X-API-Key übergeben werden.
Berechtigungen — Die erforderlichen API-Key-Berechtigungen sind in der Dokumentation des jeweiligen Endpunkts angegeben. Es gibt 3 Anforderungsstufen:
All
— Alle angezeigten Berechtigungen sind erforderlich.
Any
— Eine beliebige der angezeigten Berechtigungen ist erforderlich.
Single
— Nur die eine angezeigte Berechtigung ist erforderlich.
Zugriffsprotokollierung — Zugriffe auf die API, die aufgrund fehlender Berechtigungen oder einer nicht übereinstimmenden IP-Adresse eingeschränkt wurden, werden im Systemlog protokolliert.
JSON-Struktur — Alle Daten werden in JSON-Objekten innerhalb eines übergeordneten Schlüssels data gesendet und empfangen. Ausnahmen gelten für binäre Antworten oder Anfragen.
Zeichenkodierung — Alle Daten müssen UTF-8-kodiert sein.
Zeitstempel — Zeitstempel müssen als lokale Zeit übergeben werden. Unter Tools ist ein Timestamp-Konverter zu finden.
Datumsformat — Daten werden im PHP-Format Date_ATOM übermittelt (Beispiel: 2005-08-15T15:52:01+00:00).
Pagination — Paginierte Antworten enthalten die Daten in der Eigenschaft items. Die Pagination-Informationen werden in den Eigenschaften totalItemCount, pageStart (Offset) und pageSize (Standard: 10, max. 1000) übermittelt.

REST API for the Nimbuscloud platform.

Authentication
The API key must be provided in the X-API-Key header on every request.

Permission requirements
Every endpoint documents one of three permission requirement levels:

Single — the single shown permission is required.
All — every shown permission is required.
Any — at least one of the shown permissions is required.
Access that is rejected because of missing permissions or an IP address mismatch is recorded in the system log.

Envelope conventions
All JSON requests and responses wrap their payload in a top-level data property. Binary requests and responses are exempt.

Dates are exchanged in PHP DATE_ATOM format (ISO 8601 with timezone), e.g. 2005-08-15T15:52:01+00:00.

Pagination
Paginated responses use:

items — the page of records (inside data)
totalItemCount — total number of records matching the query
pageStart — offset of the first returned record (defaults to 0)
pageSize — number of records per page (defaults to 10, max 1000)
The corresponding request query parameters pageStart and pageSize follow the same defaults and limits.
